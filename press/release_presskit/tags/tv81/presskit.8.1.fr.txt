Deux des nombreuses fonctionnalités avancées ajoutées à PostgreSQL sont :

Two-Phase Commit (2PC): longtemps demandé par les applications WAN et les
centres de données hétérogènes utilisant PostgreSQL, cette fonctionnalité permet
des transactions compatibles ACID au travers de serveurs éloignés.

Rôles: Â PostgreSQL dispose maintenant de la notion de rôles dans la base de
données, simplifiant ainsi la gestion de grandes bases d'utilisateurs avec des
droits complexes sur les bases.

La version 8.1 inclus également quelques fonctionnalités qui amélioreront les
performances pour les applications OLTP nécessitant de gros volumes et pour les
gros entrepots de données.

Amélioration de performance en multiprocesseur (SMP) : Â le gestionnaire de
tampons de la version 8.1 a été amélioré afin de s'adapter de manière
quasi-linéaire au nombre de processeurs, amenant ainsi des gains de performances
significatifs sur les serveur à 8 et 16 processeurs.

Bitmapscan: Â Certains index seront automatiquement contertis en bitmap en
mémoire, permettant d'obtenir des performances d'index 20x plus rapide sur des
requêtes complexes sur des grosses tables, comme on en trouve dans les fermes de
données. Le bitmapscan permet également de réduire le besoin d'index sur
plusieurs colonnes.

Partitionnage de tables Partitioning : dans la version 8.1, le plannificateur de
requête peut sélectionner les bonnes partitions d'une table pour chaque requête
(appelé exclusion de contraintes (Constraint Exclusion)) 
in version 8.1, the query planner's ability
to select the correct table partitions for each query (called
Constraint Exclusion) is expanded, making PostgreSQL's table
partitioning useful to a broader range of applications.

Shared Row Locking: PostgreSQL's "better than row-level
locking" has been improved further through the addition of
shared row locks for foreign keys.  Shared locks will improve
insert and update performance on some OLTP applications.
